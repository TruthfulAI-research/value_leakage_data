# value_leakage_data

Cached model rollouts and judge outputs for
[Value Leakage: An LLM's Answers Are Silently Shaped by Its Own Values](https://github.com/TruthfulAI-research/value_leakage).
This repository is used as the `data/` submodule of the code repository; the
analysis and plotting scripts there run from these caches without API access.

- `final_data/` — rollout and judge caches for all experiments (Donation Bet, AI Company Questions, Job Offer, Agentic Grading)
- `choosing_activities/results/` — Choosing Activities pipeline results
- `site/` — the project landing page (static HTML)
- `static/` — the rollout browser dashboard (`index.html` + `app.js` + `style.css`) and its per-(experiment, model, variant) JSON exports under `static/data/` (generated by `export_browser_data.py` in the code repository; a fixed random sample of up to 90 rollouts per cell, 270 for activities / agentic grading, ~900 MB)

## Project site (valueleakage.net)

The public site is GitHub Pages, published by
`.github/workflows/pages.yml` on every push to `main` that touches `site/`
or `static/`:

- `/` — landing page (`site/`)
- `/browser/` — the rollout browser (`static/`, served as-is)

To preview locally, serve the directory you are working on, e.g.:

```bash
(cd static && python3 -m http.server 8000)   # the browser: http://localhost:8000
(cd site && python3 -m http.server 8001)     # the landing page (its /browser link only works on the deployed site)
```
